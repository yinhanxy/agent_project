# 知识库测试语料（test-corpus）

一套**通用企业行政制度**测试文档 + 批量上传脚本 + 测试问题清单，用于验证
LangGraph Agent 这条链路：Coordinator 任务分类 → Knowledge 检索 → Task 执行
（对比 / 报告 / 申请）→ KnowledgeGap 缺口记录。

## 一、文档清单

| 文件 | 主题 | 主要用途 |
|---|---|---|
| `01-差旅与报销管理办法-2023版.md` | 报销（旧版） | 与 02 构成**新旧版对比** |
| `02-差旅与报销管理办法-2025版.md` | 报销（新版） | 与 01 构成**新旧版对比** |
| `03-考勤与请假管理制度.md` | 考勤/年假/病假/加班 | 问答、**报告生成** |
| `04-远程办公管理规定.md` | 远程办公申请/设备 | 问答、**申请生成** |
| `05-员工入职与转正管理办法.md` | 试用期/转正/年假资格 | 问答 |
| `06-IT设备与办公用品管理规定.md` | 设备领用/报修/采购 | 问答 |

> 格式说明：全部用 `.md`，后端对 Markdown 走 **semantic 切片**（按标题边界切块），
> 检索/对比质量最好。前端「知识库」页也支持手动上传这些文件。

## 二、灌库

### 方式 A：批量上传脚本（推荐）

确保 Django(8001) + 后端(8000) + Milvus + Ollama 已启动，然后：

```powershell
cd D:\source\agent\LangChain-RAG-FastAPI-Service\docs\test-corpus
# 默认 -Scope company：建/复用一个公开库再上传，全员可检索（需管理员账号）
.\upload_test_corpus.ps1 -Username <管理员账号> -Password <密码>
# 后端跑在 8010 时：加 -ApiBase http://127.0.0.1:8010
# 想灌个人库（仅上传账号可见）：加 -Scope personal
```

脚本会：登录拿 JWT → 幂等地建/复用 company 公开库「测试制度库（行政）」→ 逐个
`POST /api/kb/{kb_id}/documents` 上传。**company 库全员可检索到**，多账号测试不踩权限坑。

### 方式 B：前端手动上传

浏览器开 `http://localhost:3000` → 登录（管理员）→ 「知识库」页 →「共享知识库」Tab →
「创建知识库」选**公开**范围 → 进入该库「上传」这 6 个 `.md` 文件。
（若放「个人文档」Tab，则只有该账号能检索到，见下方权限说明。）

## 二.五、权限模型（务必先读，否则测试会失真）

检索按 `assemble_rag_filter` 过滤：

| 测试账号 | 能检索到的文档 |
|---|---|
| **管理员**（`is_admin`） | 全库可见，谁传的都能检索到 |
| **普通用户** | 自己的个人库 **+** 本部门 `dept` 库 **+** `company` 公开库 **+** 被授权库 |

由此三条铁律：

1. **个人库按 `user_id` 硬隔离**：上传账号 ≠ 聊天账号且后者非管理员 → 一条都检索不到，
   本该命中的问答会退化成"找不到依据"甚至误判知识缺口。**别把这当功能 bug。**
2. **本语料默认灌 `company` 公开库**就是为绕开第 1 条：任何账号都能检索到，干净。
3. **知识缺口记录**按 `user_id`/`dept_id` 归属：普通用户只看自己的、管理员看全部 —— 上传/提问/查看尽量用同一账号才自洽。

## 三、测试问题清单（核心）

> 前提：AI 聊天要走 LangGraph 引擎，后端启动需设 `AGENT_ENGINE=graph`
> （否则走旧的 loop 引擎，不经 Coordinator/Task/Gap 节点）。聊天页观察"步骤"
> 提示即可判断命中了哪个节点。

### 1. 知识问答 `knowledge_qa`（应命中检索并带来源引用）

- 一线城市出差住宿费每晚上限是多少？（2025 版应答 550 元）
- 工龄满 12 年每年有几天年假？（应答 10 天）
- 远程办公申请需要满足哪些条件？
- 试用期员工有带薪年假吗？（应答：无，转正后按工龄算）
- 笔记本电脑多久更换一次？（应答 4 年）

预期：步骤显示"识别任务类型 → knowledge_qa"，回答末尾有**来源引用**（filename + 命中片段 + 相似度）。

### 2. 文档对比 `document_compare`（触发 Task 节点 compare_tool）

- 对比一下差旅报销办法 2023 版和 2025 版有哪些差异？
- 新版报销制度在审批额度和报销时限上比旧版有什么变化？

预期：步骤出现"生成文档对比"，回答应列出差异点（如市内交通 80→120、住宿一线 450→550、
审批额度 5000→8000、报销时限 30→60 天、打款 5→3 个工作日）。

### 3. 报告生成 `report_generation`（触发 Task 节点 report_tool）

- 根据考勤与请假制度，生成一份请假类型与待遇的要点报告。
- 帮我整理一份公司各类假期（年假/病假/事假/婚丧假）的对照报告。

预期：步骤出现"生成报告"，回答为结构化报告（分节/列表）。

### 4. 申请生成 `document_generation`（触发 Task 节点 form_tool）

- 帮我写一份远程办公申请，事由是家中老人需要照顾，时间下周一到周三。
- 帮我起草一份病假申请，因感冒发烧请假两天。

预期：步骤出现"生成申请文本"，回答为可直接用的申请正文，且符合制度（如远程每周≤2 天的限制）。

### 5. 知识缺口 `knowledge_gap`（库里**故意没有**，应自动记缺口）

下列主题这 6 份文档里都没有，用来触发缺口记录：

- 公司的员工持股 / 股权激励计划细则是怎样的？
- 境外出差的签证费和国际机票怎么报销？
- 公司有没有补充商业医疗保险？保额多少？
- 陪产假和育儿假各有几天？

预期：步骤出现"记录知识缺口"，回答会如实告知"知识库没有明确依据，已生成待补充条目"；
随后到「我的 → 知识缺口」页应能看到新记录（管理员看全部）。

> 已知边界（见 `docs/plans/2026-06-03-knowledge-gap-5a.md` 设计）：缺口触发对"检索到弱相关文档"
> 不敏感，真正稳定触发需检索 0 命中 / 极低置信，或 Coordinator 直接判定为 knowledge_gap。
> 所以请用与制度**完全无关**的主题来测，命中率最高。
